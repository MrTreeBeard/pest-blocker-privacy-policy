Privacy Policy — Pest Blocker

Last updated: 2026-01-14
Summary: Pest Blocker does not collect, transmit, sell, or share personal data. The extension runs locally in your browser. Settings you create are stored using Chrome extension storage.

1) Data collection

Pest Blocker does not collect personal data.

The extension does not collect, log, or transmit:

  • Personally identifiable information
  • Browsing history for analytics or tracking
  • Page content (text, images, form inputs)
  • Passwords, payment data, or messages
  • IP addresses or device identifiers for profiling

2) Data use

Because Pest Blocker does not collect personal data, it does not use user data for:

  • Advertising
  • Analytics
  • Profiling
  • Tracking
  • Monetization

All functionality is performed locally to block user-specified domains and prevent unwanted popup/new-tab behavior.

3) Local storage (user-controlled)

Pest Blocker stores user-controlled configuration locally using Chrome’s extension storage APIs. This may include:

  • Blocked domain list
  • Popup allowlists (global and per-site)
  • Popup policy mode and temporary disable state
  • Optional local block log (if enabled)

This information is created by the user and is used only to provide the extension’s features. It remains on the user’s device (and may sync via Chrome Sync if the user has sync enabled in Chrome).

4) Data sharing

Pest Blocker does not share data with third parties.

No user data is sold, rented, transferred, or disclosed to advertisers, analytics providers, data brokers, or any external services.

5) Network access

Pest Blocker does not send user data to external servers. It does not use third-party analytics or advertising SDKs.

6) Permissions explanation

Pest Blocker requests the following permissions strictly to provide its core functionality:

  • declarativeNetRequest / declarativeNetRequestWithHostAccess: to block user-specified domains at the network layer.
  • host_permissions (site access): to apply user-defined rules on sites the user visits.
  • tabs: to detect and close unwanted popup/new-tab behavior, and to identify the active tab when the user selects “Block current site”.
  • storage: to save user settings, block/allow lists, and optional log preferences.

7) Remote code

Pest Blocker does not use remote code. All executable code is packaged with the extension.

8) Changes to this policy

If the extension’s functionality changes in a way that affects privacy, this policy will be updated.
